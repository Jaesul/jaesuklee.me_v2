Javascript: The Language of the Web

The most recent coding language that I've added to my toolbelt. JavaScript is my favorite language its simple to use (in my perspective), and is the primary language for the web. It’s funny because you always start out trying to optimize and learn a language that’s the perfect fit for you, but for web that decision is already made for you. Of course, there’s TypeScript and other *ahem* no name coding languages for the web (kidding) but JavaScript is the most popular and with the most support. 9.5/10 Would code with again.

CSS: It cascades alright 😉

Oh, Cascading Style Sheets, why the f are you so complicated and frustrating at times. You know, you write some CSS without looking at the output and in your head, you think you know what’s going to output, but then you look and it’s like what the f is on my screen. I had a difficult time wrapping my head around CSS especially how hard it is to align elements vertically before flexbox and grid. How the heck does that make any sense. I feel bad for developers that wrote stylesheets before this new age of markup. 5.7/10 Would code again only because it’s the only option.

Project Portfolio: Let's Get Hired!

Overview

This project was to develop my skills as a front-end developer and to showcase my experience and projects. I also leveraged my design background to create a beautiful, yet simplistic layout, and maintain visual hierarchy and usability.

Problem

I don’t have a central location to showcase my personal experiences and qualifications. I also don’t have a solid front end project under my belt.

Objective

To develop a front-end portfolio that showcases my works as a developer and designer and to exercise my design skills.

Creating Design Requirements

Guidelines for designing

Creating design requirements was an easy task since there wasn’t any user research needed. Showcasing my relevant experience simply had to be beautiful and direct the user to the most important parts. These design requirements were crucial through the design process, for every new addition I came back to these requirements to ensure that I was following the guidelines I had set out.

Clear Hierarchy

The website needs to have clear hierarchy to guide the user to the most important parts of the website. This is also necessary to enhance readability for the user.

Iterate on Design

Don’t stick to one static, keep redesigning until the desired product is created. If the whole thing needs to be thrown out, then do what’s necessary.

Receive Critique

You might think this is the hottest thing since sliced bread, but it might not be. Get critique from your design friends and the general public about what works and what doesn’t.

White Space

White space must be managed carefully to maintain readability. More white space is better than less, do not squish elements too close to each other. Chunks or sections are spaced further apart than headings to body.

Maintain Good Code Style

JavaScript must not be redundant or lack documentation. Maintain code reusability.

Explore New Methods

New technologies or stylings are necessary to develop skills as a developer. Try to use new technologies to further your breath of knowledge

Creating Wireframes

I created wireframes to start the design process, but I wasn’t very specific on element positioning, white space, or hierarchy. I wanted to create a general layout and iterate multiple times since I knew it was going to be a process of designing and redesigning. These wireframes only took about 30 minutes to draw up, then I started the development process.
